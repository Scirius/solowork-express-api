Node.js exercise, API+routing

Introduction

This is a coding exercise for WBS Coding School Full Stack Web/App Development Course.

  • PATH /users
    • Defined in usersRoutes.js
    • GET /users/ Displays all users
    • GET /users/:id Gets single user
    • PUT /users/:id Updates user (fields 'firstname' and 'lastname' are rquired)
    • DELETE /users/:id Deletes the given user
  • PATH /
    • GET / Displays static html website explanation (this)
    • GET /static Static folder for html intro
    • GET /styles Static folder for css assets
  • PATH /demo/
    • Defined in demoRoutes.js
    • GET /demo/recreate Allows you to reset demo table and sample data
  • API Testing
    • Postman (Webtool)
      offers great tools for endpoint testing with a wide variety of methods (GET, POST, PUT, DELETE, PATCH, UPDATE...)
    • Postman (Desktop App)
      the desktop app is available for Win / Mac OS / Linux
  • PostgreSQL Backend
    • Elephant SQL (PostgreSQL as a Service)
      offers a free plan for developers to host PostgreSQL databases
  • Node.js
  • Express
  • PG
  • Routing
  • SQL (PostgreSQL on Elephant SQL)
  • HTML5, CSS3, JS
  • DB Connection should be done with async/await
  • Sanitizing of query, params, headers
